## Best Practices

1. **Fork first** — Always fork before editing
2. **Follow exact format** — Match existing entries precisely
3. **Add description** — One line describing value
4. **Include badges** — Stars count (optional but recommended)
5. **PR description** — Explain why this belongs
6. **Follow up** — If no response in 7 days, comment on PR
7. **Be patient** — Some lists take months to review

## Submission Template

```markdown
## PR Title
Add [repo-name] to [list-name]

## Description
I'd like to suggest adding [repo-name] to [list-name].

[Repo] is [one-line description] that helps [target users] [solve problem].

It meets the list requirements:
- ✅ Has README with documentation
- ✅ Has tests
- ✅ Active maintenance (last release: [date])
- ✅ [Other requirements]

## Why it belongs here
[Explain the value it provides to this list's audience]

## Similar projects
- [Project 1]
- [Project 2]
```

## Tracking Guidelines

For each submission, update:
- **Status:** Not started → In progress → Submitted → Under review → Accepted/Rejected
- **Date Submitted:** When PR was created
- **Date Responded:** When maintainer responded
- **Notes:** Any feedback, requests for changes
